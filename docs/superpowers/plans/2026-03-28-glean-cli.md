# Glean CLI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a single-file bash CLI tool that orchestrates Claude Code sessions to discover open source projects, deeply analyze each one against a design spec, and synthesize cross-cutting insights into spec enhancements.

**Architecture:** Single `glean` bash script orchestrating `claude -p` sessions via session forking. Base session reads the spec once; all subsequent sessions fork from it for token cache efficiency. Parallel repo analysis with worker pool. State persisted to JSON for crash recovery.

**Tech Stack:** Bash 4.0+, Claude Code CLI (`claude`), git, jq. No other runtime dependencies. Tests use a lightweight bash test harness with mock `claude`/`git` binaries.

    test_parse_args.sh            # Tests for argument parsing
    test_logging.sh               # Tests for log functions
    test_state.sh                 # Tests for state save/load
    test_url_extraction.sh        # Tests for GitHub URL extraction from markdown
    test_convergence.sh           # Tests for critique convergence detection
    test_session.sh               # Tests for claude session orchestration
    test_parallel.sh              # Tests for worker pool
    test_main_flow.sh             # Integration tests for the full pipeline
    run_tests.sh                  # Runner — executes all test_*.sh, reports results
```

Each file's responsibility:
- **`glean`**: CLI entry point. Parses args, checks deps, orchestrates all 5 phases, manages state, handles signals. Organized as functions with a `main` at the bottom guarded by `[[ "${BASH_SOURCE[0]}" == "${0}" ]] && main "$@"` so tests can source it.
- **`tests/helpers.sh`**: `assert_equals`, `assert_contains`, `assert_exit_code`, `setup_mocks` (prepends `tests/mocks/` to PATH), `create_temp_dir`, `cleanup_temp_dir`. Tracks pass/fail counts.
- **`tests/mocks/claude`**: Reads `MOCK_CLAUDE_RESPONSE` env var or a default; echoes it as JSON. Supports `MOCK_CLAUDE_EXIT_CODE` for failure simulation.
- **`tests/mocks/git`**: Simulates `git clone` by creating the target directory with a dummy file. Supports `MOCK_GIT_FAIL=1` for failure simulation.

#!/usr/bin/env bash
# Lightweight test harness for glean

TESTS_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
PROJECT_DIR="$(cd "$TESTS_DIR/.." && pwd)"
GLEAN_SCRIPT="$PROJECT_DIR/glean"

_PASS_COUNT=0
_FAIL_COUNT=0
_TEST_TMPDIR=""

setup_mocks() {
    export PATH="$TESTS_DIR/mocks:$PATH"
}

create_temp_dir() {
    _TEST_TMPDIR="$(mktemp -d)"
    echo "$_TEST_TMPDIR"
}

cleanup_temp_dir() {
    [[ -n "$_TEST_TMPDIR" && -d "$_TEST_TMPDIR" ]] && rm -rf "$_TEST_TMPDIR"
    _TEST_TMPDIR=""
}

assert_equals() {
    local expected="$1" actual="$2" msg="${3:-}"
    if [[ "$expected" == "$actual" ]]; then
        _PASS_COUNT=$((_PASS_COUNT + 1))
    else
        _FAIL_COUNT=$((_FAIL_COUNT + 1))
        echo "  FAIL: ${msg:-assert_equals}"
        echo "    expected: $expected"
        echo "    actual:   $actual"
    fi
}

assert_contains() {
    local haystack="$1" needle="$2" msg="${3:-}"
    if [[ "$haystack" == *"$needle"* ]]; then
        _PASS_COUNT=$((_PASS_COUNT + 1))
    else
        _FAIL_COUNT=$((_FAIL_COUNT + 1))
        echo "  FAIL: ${msg:-assert_contains}"
        echo "    expected to contain: $needle"
        echo "    in: $haystack"
    fi
}

assert_not_contains() {
    local haystack="$1" needle="$2" msg="${3:-}"
    if [[ "$haystack" != *"$needle"* ]]; then
        _PASS_COUNT=$((_PASS_COUNT + 1))
    else
        _FAIL_COUNT=$((_FAIL_COUNT + 1))
        echo "  FAIL: ${msg:-assert_not_contains}"
        echo "    expected NOT to contain: $needle"
        echo "    in: $haystack"
    fi
}

assert_exit_code() {
    local expected="$1" actual="$2" msg="${3:-}"
    if [[ "$expected" == "$actual" ]]; then
        _PASS_COUNT=$((_PASS_COUNT + 1))
    else
        _FAIL_COUNT=$((_FAIL_COUNT + 1))
        echo "  FAIL: ${msg:-assert_exit_code}"
        echo "    expected exit code: $expected"
        echo "    actual exit code:   $actual"
    fi
}

assert_file_exists() {
    local filepath="$1" msg="${2:-}"
    if [[ -f "$filepath" ]]; then
        _PASS_COUNT=$((_PASS_COUNT + 1))
    else
        _FAIL_COUNT=$((_FAIL_COUNT + 1))
        echo "  FAIL: ${msg:-assert_file_exists}"
        echo "    file not found: $filepath"
    fi
}

report_results() {
    local test_name="$1"
    echo "$test_name: $_PASS_COUNT passed, $_FAIL_COUNT failed"
    return "$_FAIL_COUNT"
}
```

check_dependencies() {
    local missing=()
    command -v claude >/dev/null 2>&1 || missing+=("claude")
    command -v git >/dev/null 2>&1    || missing+=("git")
    command -v jq >/dev/null 2>&1     || missing+=("jq")

    if [[ ${#missing[@]} -gt 0 ]]; then
        echo "glean: missing required dependencies: ${missing[*]}" >&2
        echo "Install them and try again." >&2
        return 3
    fi

    # Check bash version
    if [[ "${BASH_VERSINFO[0]}" -lt 4 ]]; then
        echo "glean: requires bash 4.0+, found $BASH_VERSION" >&2
        return 3
    fi
    return 0
}

parse_args() {
    # Reset globals to defaults
    SPEC_FILE=""
    OUTPUT_DIR="$DEFAULT_OUTPUT_DIR"
    CONTINUE_MODE=false
    MODEL="$DEFAULT_MODEL"
    EFFORT="$DEFAULT_EFFORT"
    WORKERS="$DEFAULT_WORKERS"
    MAX_REPOS="$DEFAULT_MAX_REPOS"
    GREENFIELD=false
    VERBOSE=false
    PRESEEDED_REPOS=()

    while [[ $# -gt 0 ]]; do
        case "$1" in
            --help|-h)
                usage
                exit 0
                ;;
            --version)
                echo "glean $GLEAN_VERSION"
                exit 0
                ;;
            --continue)
                CONTINUE_MODE=true
                shift
                ;;
            --output-dir)
                [[ $# -lt 2 ]] && { echo "glean: --output-dir requires a value" >&2; return 2; }
                OUTPUT_DIR="$2"
                shift 2
                ;;
            --repos)
                shift
                while [[ $# -gt 0 && ! "$1" =~ ^-- ]]; do
                    PRESEEDED_REPOS+=("$1")
                    shift
                done
                ;;
            --max-repos)
                [[ $# -lt 2 ]] && { echo "glean: --max-repos requires a value" >&2; return 2; }
                MAX_REPOS="$2"
                shift 2
                ;;
            --model)
                [[ $# -lt 2 ]] && { echo "glean: --model requires a value" >&2; return 2; }
                MODEL="$2"
                shift 2
                ;;
            --effort)
                [[ $# -lt 2 ]] && { echo "glean: --effort requires a value" >&2; return 2; }
                EFFORT="$2"
                shift 2
                ;;
            --workers)
                [[ $# -lt 2 ]] && { echo "glean: --workers requires a value" >&2; return 2; }
                WORKERS="$2"
                shift 2
                ;;
            --greenfield)
                GREENFIELD=true
                shift
                ;;
            --verbose)
                VERBOSE=true
                shift
                ;;
            --*)
                echo "glean: unknown option: $1" >&2
                return 2
                ;;
            *)
                if [[ -z "$SPEC_FILE" ]]; then
                    SPEC_FILE="$1"
                else
                    echo "glean: unexpected argument: $1" >&2
                    return 2
                fi
                shift
                ;;
        esac
    done

    if [[ -z "$SPEC_FILE" ]]; then
        echo "glean: spec file is required" >&2
        return 2
    fi
}
```

run_parallel_analysis() {
    local pids=()
    local repos_started=0
    local repos_total=${#REPO_URLS[@]}
    local failed_repos=()

    log_info "Phase 4: Analyzing $repos_total repos with $WORKERS workers..."
    save_state "deep-dives" ""

    while [[ $repos_started -lt $repos_total ]] || [[ ${#pids[@]} -gt 0 ]]; do
        # Launch workers up to WORKERS limit
        while [[ ${#pids[@]} -lt $WORKERS && $repos_started -lt $repos_total ]]; do
            local url="${REPO_URLS[$repos_started]}"
            repos_started=$((repos_started + 1))
            (
                analyze_repo "$url"
            ) &
            pids+=($!)
        done

        # Wait for any one to finish
        if [[ ${#pids[@]} -gt 0 ]]; then
            local new_pids=()
            for pid in "${pids[@]}"; do
                if kill -0 "$pid" 2>/dev/null; then
                    new_pids+=("$pid")
                else
                    wait "$pid" || failed_repos+=("$pid")
                fi
            done
            pids=("${new_pids[@]}")
            # Brief pause to avoid busy-waiting
            [[ ${#pids[@]} -gt 0 ]] && sleep 1
        fi
    done

    if [[ ${#failed_repos[@]} -gt 0 ]]; then
        log_error "${#failed_repos[@]} repo(s) failed analysis"
        return 1
    fi
    return 0
}

deep analyses of open source projects evaluated against the spec
you read at the start of this conversation.

Synthesize the cross-cutting insights into concrete spec
enhancement recommendations. Focus on:
- Architectural patterns that appear across multiple projects
- Structural advantages the spec has that should be emphasized
- Gaps or blind spots the spec should address
- Ideas that can be combined or reimagined

EOF

run_synthesis() {
    log_info "Phase 5: Synthesizing cross-cutting insights..."

    local prompt
    prompt=$(build_synthesis_prompt)
    local allowed_tools="Read,Glob,Grep,Write"

    local session_id
    session_id=$(run_claude_fork "$BASE_SESSION_ID" "$prompt" "$allowed_tools")

    if [[ -z "$session_id" || "$session_id" == "null" ]]; then
        log_error "Synthesis session failed"
        return 1
    fi

    log_session "synthesis" "Synthesis session: $session_id"

    # Update state
    local tmp
    tmp=$(mktemp)
    jq '.synthesis = "complete"' "$OUTPUT_DIR/.glean-state.json" > "$tmp"
    mv "$tmp" "$OUTPUT_DIR/.glean-state.json"

    log_info "Synthesis complete"

    if [[ ! -f "$OUTPUT_DIR/synthesis.md" ]]; then
        log_error "Synthesis did not produce synthesis.md"
        return 1

```markdown
# Glean

Automated spec enhancement through open source intelligence.

Glean reads a design spec, discovers relevant open source projects, deeply analyzes each one through the lens of the spec, and synthesizes cross-cutting insights into concrete spec enhancements.

## Requirements

- bash 4.0+
- [Claude Code](https://docs.anthropic.com/en/docs/claude-code) CLI
- git
- jq

## Install

```bash
curl -fsSL https://raw.githubusercontent.com/{owner}/glean/main/install.sh | bash
```

Or clone and use directly:

```bash
git clone https://github.com/{owner}/glean.git
cd glean
./glean --help
```

## Usage

```
glean [options] <spec-file>

Options:
  --continue            Resume after editing the discovery list or after interruption
  --output-dir <dir>    Output directory (default: ./gleanings)
  --repos <urls...>     Pre-seed specific repo URLs (skips discovery)
  --max-repos <n>       Maximum repos to deep-dive (default: 10)
  --model <model>       Claude model (default: opus)
  --effort <level>      Thinking effort: medium, high, max (default: high)
  --workers <n>         Parallel repo analyses (default: 3)
  --greenfield          Forbid reading local codebase during analysis
  --verbose             Stream Claude output to terminal
```

### Typical workflow

```bash
# 1. Discover relevant projects
glean design-docs/my-spec.md

# 2. Review and edit the candidate list
vim gleanings/reference-projects.md

# 3. Run deep analysis + synthesis
glean --continue design-docs/my-spec.md
```

### Skip discovery with known repos

```bash
glean design-docs/my-spec.md \
  --repos https://github.com/m-bain/whisperx https://github.com/WyattBlue/auto-editor
```

## Output

```
gleanings/
  reference-projects.md    # Discovery index (human-editable)
  synthesis.md             # Cross-repo spec enhancements
  analysis-whisperx.md     # Per-repo deep analysis
  analysis-auto-editor.md
  ...
```

## How it works

1. **Spec ingestion** — Creates a base Claude session that reads your spec
2. **Discovery** — Forks from base, searches GitHub/web for related projects
3. **Human checkpoint** — You review and edit the candidate list
4. **Parallel deep-dives** — For each repo: initial analysis, deeper analysis, structural inversion, then iterative critique until convergence
5. **Synthesis** — Cross-cutting insights distilled into spec enhancements

All sessions fork from the base, sharing cached spec context for efficiency.

## Crash recovery

Glean persists pipeline state to `.glean-state.json`. If interrupted, `--continue` resumes from where it left off — completed repos are skipped, in-progress repos resume at their current step.
